核心秘密：一个 While 循环

The Core Secret: A While Loop

没有复杂的 DAG，没有规划器，就是一个循环。

No complex DAGs, no planners — just one loop.

整个系统的心脏只有一行伪代码：while (模型还想调用工具) { 执行工具 → 把结果喂回模型 }。模型自己决定下一步做什么，系统只负责执行。这个设计看起来简单，但背后有 7 种不同的"继续循环"原因和 5 层消息压缩机制。

The entire system's heart is one line of pseudocode: while (model wants to call tools) { execute tool → feed result back }. The model decides what to do next; the system just executes. This looks simple, but behind it are 7 different "continue" reasons and 5 layers of message compression.

为什么不用 DAG/规划器？Why not DAGs/planners? 因为让模型自己决定比预设流程更灵活。模型走错了可以自我纠正，而 DAG 一旦定义就固定了。代价是可能多花几次 API 调用，但换来了极高的灵活性。 Because letting the model decide is more flexible than preset workflows. The model can self-correct mistakes, while DAGs are fixed once defined. The trade-off is potentially more API calls, but extreme flexibility.

省钱秘诀：92% 缓存命中率

Cost Saving: 92% Cache Hit Rate

每次 API 调用都要发送完整的 System Prompt，但只需付 10% 的钱。

Every API call sends the full System Prompt, but you only pay 10% of the cost.

System Prompt 被分成两个区域：静态区（身份、规则、工具说明）从不变化，跨所有用户共享缓存；动态区（环境信息、记忆、MCP 配置）每次会话不同。一条分界线 __DYNAMIC_BOUNDARY__ 把它们隔开。

The System Prompt is split into two zones: the static zone (identity, rules, tool descriptions) never changes and is cached globally across all users; the dynamic zone (environment, memory, MCP config) varies per session. A boundary marker __DYNAMIC_BOUNDARY__ separates them.

10.2% 灾难：The 10.2% disaster: Agent 列表曾嵌在工具描述中（属于静态区）。MCP 连接一变，列表就变，整个缓存失效。这一个问题消耗了平台 10.2% 的缓存创建 token。修复：移到消息附件中。 The Agent list was embedded in tool descriptions (static zone). When MCP connections changed, the list changed, busting the entire cache. This single issue consumed 10.2% of fleet cache creation tokens. Fix: moved to message attachments.

设计哲学是 Fail-Closed：任何无法证明安全的命令都需要你确认。系统用 tree-sitter 做 AST 解析（白名单制），shell-quote 做备用。代码注释里引用了 5 个 HackerOne 安全报告——都是真实的渗透测试发现。

Design philosophy: Fail-Closed. Any command that can't be proven safe requires your confirmation. Uses tree-sitter for AST parsing (whitelist-based), shell-quote as fallback. Code comments reference 5 HackerOne reports — real penetration test findings.

Fork Agent: 最精妙的设计——子 Agent 继承父 Agent 的完整对话上下文，实现字节级 prompt 缓存共享。父子只在最后一条指令处"分叉"，共享前缀部分的所有缓存费用。 The most elegant design — child inherits parent's full conversation context, achieving byte-level prompt cache sharing. Parent and child diverge only at the final directive, sharing all cache costs for the prefix.
